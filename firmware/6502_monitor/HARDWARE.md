# VeraX16 PBI — Hardware Reference

## 1. Overview

The ESP32 interfaces the Atari 800XL PBI (Parallel Bus Interface) to:

- serve a 2 KB ROM image at **$D800–$DFFF** (PBI device ROM)
- respond to register reads/writes at **$D100–$D1FF** (D1xx page)
- forward VERA chip accesses ($D100–$D11F) via **DEV_SEL_N**
- control the Atari floating-point ROM via **EXTSEL_N**

All 5 V Atari signals pass through **TXS0108E** bidirectional level translators
before reaching the 3.3 V ESP32 GPIOs.  The $D800–$DFFF chip-select is
generated in hardware by a **74HC138** 3-to-8 decoder.

---

## 2. Atari 6502 Bus Timing

### Clock source

| Parameter | Value |
|---|---|
| NTSC colour crystal | 14.31818 MHz |
| CPU (SALLY) clock | 14.31818 ÷ 8 = **1.7897 MHz** |
| Full bus cycle | ≈ **558.7 ns** |
| PHI2 HIGH period | ≈ **279 ns** |
| PHI2 LOW period | ≈ **279 ns** |

### Key 6502 timing parameters (65C02 / SALLY, worst-case)

| Parameter | Symbol | Value | Notes |
|---|---|---|---|
| Address valid after PHI2↓ | tADS | ≤ 300 ns | Address bus stable early in PHI2-LOW |
| Address hold after PHI2↓ | tAH | ≥ 10 ns | |
| **Read data setup before PHI2↓** | **tDSR** | **≥ 100 ns** | **Critical — ROM must drive bus in time** |
| Read data hold after PHI2↓ | tDHR | ≥ 10 ns | |
| Write data valid after PHI2↑ | tMDS | ≤ 225 ns (typ. 80–100 ns) | CPU drives data ~80 ns after PHI2↑ |
| Write data hold after PHI2↓ | tDHW | ≥ 10 ns | |

### Read cycle window

```
         PHI2
          ___________
_________|           |___________
         ^           ^
         PHI2↑       PHI2↓
         |           |
         |<- 279 ns ->|
         |           |
         |           |--- tDSR = 100 ns ---^
         |                                 |
         |<-- 179 ns: valid drive window -->|
         |
         t=0: ESP32 wakes from spin-wait
```

The ESP32 must assert valid read data within the first **179 ns** of PHI2-HIGH.

## 3. ESP32 Timing Budget

All time-critical code resides in IRAM.  ESP32 @ 240 MHz → 1 cycle ≈ 4.17 ns.

| Phase | Cycles | Time | Cumulative |
|---|---|---|---|
| Spin-wait detects PHI2↑ | 1–3 | ≤ 13 ns | ≤ 13 ns |
| Decode R/W_, D1XX_N, ROM_SEL_N | ~6 | ~25 ns | ~38 ns |
| Read GPIO_IN1 (A0–A5) | ~2 | ~8 ns | ~46 ns |
| `decode_addr()` (A0–A10) | ~18 | ~75 ns | ~121 ns |
| Sub-range decode (is_rom, is_vera…) | ~6 | ~25 ns | ~146 ns |
| `bus_drive()` incl. LUT lookup | ~7 | ~29 ns | **~175 ns** |
| **Margin before tDSR deadline** | **~1** | **~4 ns** | — |

> ⚠ **The margin is extremely tight (~4 ns nominal).**
> Any cache miss caused by code or data NOT in IRAM adds ≥ 40 ns per access
> and will produce read errors on the Atari.
>
> Every function and array accessed from `MonitorTask` **must** carry `IRAM_ATTR`:
> - `MonitorTask` itself
> - `decode_addr()`, `decode_data()`, `bus_drive()`, `bus_release()`
> - `lut_drive[]`, `int_regs[]`
> - **`pbi_driver[]`** (the 2 KB ROM image — generated by the Makefile)

### Write cycle timing

The 6502 guarantees write data valid within 225 ns of PHI2↑ (typically 80–100 ns).
The firmware re-reads `GPIO.in` after the full decode phase (~200 ns post-PHI2↑),
which is safely within the write-data valid window.

### DEV_SEL_N (VERA chip CS) latency

DEV_SEL_N is asserted ~200 ns after PHI2↑ (after the full decode path).
If the VERA chip requires CS earlier, move this decode to external
combinatorial logic (GAL/CPLD) fed directly from the address bus.

## 4. Signal Directory

| Signal | Direction | Voltage | Description |
|---|---|---|---|
| D0–D7 | ↔ bidirectional | 5 V ↔ 3.3 V | 6502 data bus |
| A0–A10 | Atari → ESP32 | 5 V → 3.3 V | Address bus (11 bits used) |
| PHI2 | Atari → ESP32 | 5 V → 3.3 V | CPU clock phase 2 |
| R/W_ | Atari → ESP32 | 5 V → 3.3 V | HIGH = read, LOW = write |
| D1XX_N | Atari → ESP32 | 5 V → 3.3 V | Active-LOW: $D1xx page selected (ext. decoder) |
| ROM_SEL_N | 74HC138 → ESP32 | 5 V → 3.3 V | Active-LOW: $D800–$DFFF (74HC138 Y7) |
| EXTSEL_N | ESP32 → Atari | 3.3 V → 5 V | Active-LOW: disables Atari floating-point ROM |
| DEV_SEL_N | ESP32 → VERA | 3.3 V | Active-LOW: VERA chip select |

### Boot-time warnings

> ⚠ **GPIO 0 (EXTSEL_N)**: if held LOW during power-on reset, the ESP32 enters
> bootloader mode.  Add a **10 kΩ pull-up** to 3.3 V on the ESP32 side of U3
> so the line cannot be pulled LOW by the Atari before the firmware takes control.
>
> ⚠ **GPIO 3 (DEV_SEL_N)**: this is UART0 RX.  The firmware disables serial RX
> with `Serial.begin(115200, SERIAL_8N1, -1, 1)`.  Do not reconnect a USB-UART
> adapter to GPIO 3 while the Atari is powered.
>
> ⚠ **GPIO 12**: internal strapping — selects 1.8 V flash VDD if HIGH at boot.
> The address line A9 must be LOW (or floating with internal pull-down active)
> during ESP32 reset.
>
> ⚠ **GPIO 34, 35, 36, 39**: hardware input-only pins on the ESP32.  They have
> no output driver and no internal pull resistors — this is correct for
> address bus inputs.

## 6. TXS0108E Level Shifter Wiring

Three TXS0108E chips translate between the 5 V Atari bus (side B) and the
3.3 V ESP32 (side A).

### Common wiring (apply to U1, U2, U3)

| TXS0108E pin | Connection | Comment |
|---|---|---|
| Pin 1 (VCCA) | 3.3 V | ESP32 supply |
| Pin 9 (GND) | GND | Common ground |
| Pin 10 (OE) | 3.3 V | Always-enabled |
| Pin 20 (VCCB) | 5 V | Atari / 74HC138 supply |

Place a **100 nF** ceramic capacitor from VCCA→GND and from VCCB→GND,
as close as possible to each chip.

---

### U1 — Data bus D0–D7 (bidirectional)

The TXS0108 auto-detects direction each transition: the ESP32 drives side A
during read cycles; the 6502 drives side B during write cycles.

| Channel | A side (GPIO, 3.3 V) | B side (Atari, 5 V) |
|---|---|---|
| A1 / B1 | GPIO 18 — D0 | Atari D0 |
| A2 / B2 | GPIO 19 — D1 | Atari D1 |
| A3 / B3 | GPIO 21 — D2 | Atari D2 |
| A4 / B4 | GPIO 22 — D3 | Atari D3 |
| A5 / B5 | GPIO 23 — D4 | Atari D4 |
| A6 / B6 | GPIO 25 — D5 | Atari D5 |
| A7 / B7 | GPIO 26 — D6 | Atari D6 |
| A8 / B8 | GPIO 27 — D7 | Atari D7 |

---

### U2 — Address bus A0–A7 (Atari → ESP32)

| A6 / B6 | GPIO 39 — A5 | Atari A5 |
| A7 / B7 | GPIO 16 — A6 | Atari A6 |

### U3 — Address A8–A10, control signals, EXTSEL_N

Seven channels are Atari → ESP32 (B dominant); one channel (A8/B8) is
ESP32 → Atari (A dominant). The TXS0108 auto-direction detection handles
this correctly because EXTSEL_N is driven as an **Open-Drain** output by the ESP32,
while the Atari signals are always driven from the B side.

| Channel | A side (GPIO, 3.3 V) | B side (5 V) | Direction | Source |
|---|---|---|---|---|
| A1 / B1 | GPIO 14 — A8 | Atari A8 | B → A | Atari |
| A2 / B2 | GPIO 12 — A9 | Atari A9 | B → A | Atari |
| A3 / B3 | GPIO 13 — A10 | Atari A10 | B → A | Atari |
| A4 / B4 | GPIO 2 — PHI2 | Atari PHI2 | B → A | Atari |
| A5 / B5 | GPIO 15 — R/W_ | Atari R/W_ | B → A | Atari |
| A6 / B6 | GPIO 5 — D1XX_N | External decoder | B → A | External |
| A7 / B7 | GPIO 4 — ROM_SEL_N | 74HC138 Y7 | B → A | 74HC138 |
| A8 / B8 | GPIO 0 — EXTSEL_N | Atari EXSEL | **A → B (OD)** | **ESP32** |

## 7. 74HC138 Address Decoder Wiring

The 74HC138 3-to-8 decoder generates **ROM_SEL_N** (active LOW) whenever
the 6502 addresses the $D800–$DFFF range (2 KB PBI ROM area).

The chip runs entirely at **5 V**; its Y7 output feeds **U3 B7**, which level-shifts
it to 3.3 V for GPIO 4.

### Pin connections

| 74HC138 pin | Name | Connected to | Voltage |
|---|---|---|---|
| Pin 1 (A) | Input A (LSB) | Atari **A11** | 5 V |
| Pin 2 (B) | Input B | Atari **A12** | 5 V |
| Pin 3 (C) | Input C (MSB) | Atari **A15** | 5 V |
| Pin 4 (G2A, active LOW) | Enable G2A | Atari **A13** | 5 V |
| Pin 5 (G2B, active LOW) | Enable G2B | **GND** | — |
| Pin 6 (G1, active HIGH) | Enable G1 | Atari **A14** | 5 V |
| Pin 7 (Y7) | Output Y7 | **U3 B7** → GPIO 4 | 5 V |
| Pin 8 (GND) | Ground | GND | — |
| Pin 9 (Y6) | Output Y6 | (optional — $D000–$D7FF) | 5 V |
| Pin 16 (VCC) | Supply | **5 V** | 5 V |

Pins Y0–Y5, Y6 (unused outputs) should be left open or connected to
downstream circuitry as needed for MPD/multi-device decoding.

### Decode logic

Y7 is driven LOW when **all** conditions are met simultaneously:

| Condition | Signal | Required | Maps to |
|---|---|---|---|
| G1 = HIGH | A14 | 1 | enabled |
| G2A = LOW | A13 | 0 | enabled (active-low) |
| G2B = LOW | GND | 0 | always enabled |
| C = HIGH | A15 | 1 | selects upper half |
| B = HIGH | A12 | 1 | selects D/E/F page group |
| A = HIGH | A11 | 1 | selects $D800 sub-range |

Combined: **A15=1, A14=1, A13=0, A12=1, A11=1 → $D800–$DFFF** ✓

### All output ranges (for reference)

| Output | Active when | Address range | Use |
|---|---|---|---|
| Y0 | A15=0, A14=1, A13=0, A12=0, A11=0 | $4000–$47FF | — |
| Y1 | A15=0, A14=1, A13=0, A12=0, A11=1 | $4800–$4FFF | — |
| Y2 | A15=0, A14=1, A13=0, A12=1, A11=0 | $5000–$57FF | — |
| Y3 | A15=0, A14=1, A13=0, A12=1, A11=1 | $5800–$5FFF | — |
| Y4 | A15=1, A14=1, A13=0, A12=0, A11=0 | $C000–$C7FF | — |
| Y5 | A15=1, A14=1, A13=0, A12=0, A11=1 | $C800–$CFFF | — |
| Y6 | A15=1, A14=1, A13=0, A12=1, A11=0 | $D000–$D7FF | MPD / spare |
| **Y7** | A15=1, A14=1, A13=0, A12=1, A11=1 | **$D800–$DFFF** | **ROM_SEL_N** |

> Note: the 74HC138 is enabled only when A14=1 AND A13=0; addresses outside
> this combination (e.g. A13=1: $E000–$FFFF) leave all outputs HIGH (inactive).

## 8. Decoupling and Layout Recommendations

- Place **100 nF** ceramics on every VCC pin of U1, U2, U3, and the 74HC138.
- Route the data bus traces (D0–D7) from U1 to the ESP32 as short as possible —
  these carry the highest transient currents during bus drive.
- Add a **10 kΩ pull-up** from GPIO 0 (EXTSEL_N, A side of U3) to 3.3 V so
  the line cannot be pulled LOW by the Atari before the firmware initialises.
- Consider a **33 Ω series resistor** on the PHI2 line (A side of U3, between
  U3 and GPIO 2) to damp ringing at the 240 MHz sampling rate.
- GPIO 12 (A9) must be LOW or floating during ESP32 reset.  If the Atari is
  powered at the same time, ensure A9 is not asserted HIGH at boot.
